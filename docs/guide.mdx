## Overview

React 18 + Vite portfolio with markdown-driven projects and blog posts. Routing via Wouter, rendering via Marked, styling via inline styles and design tokens in `src/tokens.js`.

<Note>
  This is a static React app, not Next.js. Production output lives in `dist/`.
</Note>

## Quick start

<Steps>
  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>
  <Step title="Start the dev server">
    ```bash
    npm run dev
    ```
  </Step>
  <Step title="Open the site">
    Visit [http://localhost:3000](http://localhost:3000).
  </Step>
</Steps>

### Routes

| Route | Page |
| ----- | ---- |
| `/` | Project grid with tag filters |
| `/about` | About, education, principles |
| `/blog` | Blog index |
| `/blog/:slug` | Individual blog post |
| `/projects/:id` | Project case study |

## Project structure

```
artistic_portfolio/
├── src/
│   ├── components/     # Nav, Tile, MdView, EduCard, Rule, Hairline
│   ├── pages/          # Home, About, Project, Blog, BlogPost

│   ├── data/           # projects.js, content.js
│   └── tokens.js       # Colors, typography, spacing
├── public/images/      # Static assets
├── docs/               # Mintlify documentation (this file)
├── vite.config.js
└── package.json
```

## Adding content

### New project

<Steps>
  <Step title="Add metadata">
    Edit `src/data/projects.js`:

    ```javascript
    {
      id: 'my-project',
      title: 'My Project',
      description: 'Brief description',
      tag: 'Web App',
      year: 2024,
      featured: false,
      order: 7,
    }
    ```
  </Step>
  <Step title="Write the case study">
    Create `src/content/projects/my-project.md`. The filename must match the `id`.

    ```markdown
    # My Project

    ## The problem

    ...

    ## The solution

    ...
    ```
  </Step>
  <Step title="Add images">
    Place files in `public/images/` and reference them in markdown:

    ```markdown
    ![Screenshot](./images/my-project-screenshot.png)
    ```
  </Step>
</Steps>

Markdown files under `src/content/projects/` are auto-loaded via `import.meta.glob` in `src/data/content.js`.

### New blog post

Create `src/content/blog/my-post.md`:

```markdown
---
title: My Blog Post
date: 2024-03-20
excerpt: Brief summary
---

# My Blog Post

Content here...
```

### About page

Edit `src/content/about.md`.

### Markdown image syntax

| Syntax | Effect |
| ------ | ------ |
| `![Caption](path)` | Centered image with caption |
| `![full:Caption](path)` | Full-width, no caption |
| `![half:Caption](path)` | Half-width centered |

## Customization

### Colors and layout

Edit `src/tokens.js`:

```javascript
export const colors = {
  p0: '#FAF7F2',   // base paper
  p1: '#F3EDE2',   // card surface
  // ...
};

export const spacing = {
  maxWidth: '900px',
  readingWidth: '640px',
  pagePadding: '2rem',
};
```

### Typography

Fonts: **Cormorant Garamond** (display/body) and **DM Sans** (UI). Update the Google Fonts link in `index.html` and font families in `src/tokens.js`.

### Analytics

Copy `.env.example` to `.env` and set `VITE_GOATCOUNTER_CODE` to enable GoatCounter. Leave unset to disable.

## Build and deploy

### Production build

```bash
npm run build
npm run preview   # optional local preview of dist/
```

### GitHub Pages

Push to `main`. The workflow in `.github/workflows/deploy.yml` builds and deploys automatically. Set `VITE_GOATCOUNTER_CODE` as a repo variable if you use analytics.

### GitLab Pages

`.gitlab-ci.yml` builds on the default branch and publishes `dist/` as `public/`. Set `VITE_GOATCOUNTER_CODE` in CI/CD variables.

### Other hosts

Deploy the `dist/` folder to any static host (Vercel, Netlify, etc.). Build command: `npm run build`. Output directory: `dist`. Vite uses `base: './'` so the build works under subpaths.

## Design reference

**Palette:** paper tones (`#FAF7F2`, `#F3EDE2`), warm browns (`#9C7E5A`, `#6B5240`), moss green accents (`#7A8C6A`).

**Principles:**
1. Work with the grain — technology should feel natural
2. Remove until it breaks — every element earns its place
3. Show the seams — transparency builds trust

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 already in use">
    Change the port in `vite.config.js`:

    ```javascript
    server: { port: 3001 }
    ```
  </Accordion>
  <Accordion title="Project page is empty">
    Ensure the markdown filename matches the `id` in `src/data/projects.js`.
  </Accordion>
  <Accordion title="Images not showing">
    Store images in `public/images/` and reference them as `./images/filename.jpg` in markdown.
  </Accordion>
</AccordionGroup>
